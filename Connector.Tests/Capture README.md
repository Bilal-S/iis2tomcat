# Packet Capture Test Files

## Overview

This directory contains test assets for capturing and validating raw AJP13 packet bytes.

| Directory | Description |
|-----------|-------------|
| **`jsp/`** | JSP test pages for Tomcat 11 — generates specific response scenarios (17 files) |
| **`cfm/`** | Lucee/CFML test pages — generates specific response scenarios (15 files) |
| **`captured/TomcatCapture/`** | Pre-generated `.pak` files from Tomcat 11 via JSP pages (294 files) |
| **`captured/OldCapture/`** | Legacy `.pak` captures from earlier testing (104 files) |

## Pre-Generated Captured Files

Captured `.pak` files are already available and checked into the repository — **no capture step is needed to run the tests**.

- **`captured/TomcatCapture/`** — 294 files captured from Tomcat 11 AJP responses. Generated by running `capture_tomcat.js` against JSP test pages deployed to Tomcat's webapps/ROOT. Includes `.pak` (raw wire bytes) and `.pak.meta` (metadata sidecar) for each packet. Covers all JSP test scenarios: simple text, binary of various sizes, multi-chunk responses, POST handling, status codes, headers, and a 1–15 byte padding boundary sweep.
- **`captured/OldCapture/`** — 104 files from earlier Lucee/CFML-based testing. Retained for historical comparison and cross-implementation validation.

These files are consumed directly by tests:
- `GetUserDataBytesPakReplayTests` — replays TomcatCapture PAKs through `TomcatSendBodyChunk.GetUserDataBytes()`
- `PacketCaptureFidelityTests` — validates capture byte-fidelity (generates its own temp files)
- `TomcatPacketTests` — parses PAK headers and validates packet structure

## JSP Test Pages (`jsp/`)

JSP pages generate precise AJP response patterns for Tomcat 11 testing. They use `getOutputStream()` for exact byte control (no JSP Writer whitespace issues) and `buffer="none"` to prevent Tomcat response buffering.

### How to Deploy

Copy the `jsp/` directory contents to Tomcat's web root:
```
cp Connector.Tests/packets/jsp/*.jsp /path/to/tomcat/webapps/ROOT/
```

### JSP Test Scenarios

| File | Scenario | Body Size | Notes |
|------|----------|-----------|-------|
| `test_get_simple.jsp` | GET, simple HTML text | 12B | "Hello World!", padding=0 |
| `test_get_binary.jsp` | GET, base64-encoded PNG | 67B | Minimal 1×1 PNG, padding=1 |
| `test_get_binary_odd.jsp` | GET, 3 raw bytes | 3B | 0xFF 0xFE 0xFD, padding=1 |
| `test_get_binary_singlechunk.jsp` | GET, sequential binary | 5000B | Pattern 0x01..0xFE, single chunk |
| `test_get_binary_9000.jsp` | GET, sequential binary | 9000B | Multi-chunk (8184+816) |
| `test_get_binary_large.jsp` | GET, large PNG from file | ~20KB | 3-chunk assembly, reads playstore-icon.png |
| `test_get_text_odd.jsp` | GET, 5-char text | 5B | "ABCDE", padding=3 (max padding) |
| `test_get_text_size4.jsp` | GET, 4-char text | 4B | "ABCD", padding=0 |
| `test_get_large_response.jsp` | GET, repeated byte | 16384B | 0x58 ('X') × 16384, multi-chunk |
| `test_get_sized.jsp` | GET, parametric size | 1–15B | `?size=N` — padding boundary sweep |
| `test_headers_multi.jsp` | GET, multiple headers | 10B | Tests Set-Cookie, custom headers |
| `test_status_codes.jsp` | GET, parametric status | varies | `?code=200/301/404/500` |
| `test_post_small.jsp` | POST, small form data | varies | Echoes field1, field2 |
| `test_post_chunked.jsp` | POST, chunked transfer | varies | 15000-byte body |
| `test_post_large.jsp` | POST, large form data | varies | 20000-byte field |
| `test_post_binary.jsp` | POST, base64 PNG round-trip | 67B | Decodes base64, returns raw PNG |

### Key Design Patterns

- **Sequential byte pattern**: `(i % 254) + 1` — produces 0x01..0xFE repeating. Any corruption (truncation, padding leakage, byte shifts) is immediately detectable.
- **`buffer="none"`**: Prevents Tomcat from coalescing small responses into a single buffer, ensuring each JSP produces the exact chunk boundaries expected.
- **`getOutputStream().flush()`**: Forces immediate write to the AJP response stream.

## Capturing New Packets

If you need to re-capture (e.g., after a Tomcat version change or connector code change):

### Automated (recommended)

Use the provided Node.js script to hit every JSP test page automatically:

```bash
node Connector.Tests/packets/capture_tomcat.js
```

**Prerequisites:**
- BonCode `LogLevel` set to `5` in your AJP configuration (app.config or web.config)
- IIS running with the BonCode AJP connector
- Tomcat 11 running with JSP test pages deployed to webapps/ROOT/

**What the script does:**
- Sends ~30 sequential requests (GET tests, 4 status code variations, sized sweep 1–15, POST tests) to `http://localhost/`
- Waits 500ms between requests to prevent packet interleaving
- Validates response byte patterns and sizes in real-time
- Saves `.response` files to `captured/TomcatCapture/` for offline comparison
- Prints PASS/FAIL for each test with detailed diagnostic messages

**Output:** `.pak` and `.pak.meta` files appear in BonCode's configured log directory (default: `C:\Sites\AJP13\BIN\`). Move or copy them to `captured/TomcatCapture/` to update the test data.

### Manual

1. Set `LogLevel` to `5` in your BonCode AJP configuration
2. Make requests through IIS → BonCode → Tomcat to each `.jsp` test page
3. Raw packet files (`.pak` + `.pak.meta`) are written to the configured log directory
4. Copy selected `.pak` files to `captured/TomcatCapture/`

### CFM-Based Capture (Legacy)

For Lucee/CFML testing, use the CFM pages in `cfm/` with the older capture script:

```bash
node Connector.Tests/packets/capture.js
```

This follows the same pattern but targets `.cfm` pages on Lucee.

## File Format

### `.pak` files

Raw wire-level bytes of a single AJP13 packet:
```
[magic: 0x12 0x34] [length: 2B big-endian] [type: 1B] [data: NB] [padding: 0-3B]
```

### `.pak.meta` files

Human-readable metadata sidecar:
```
Timestamp: 2026-03-21 13:45:00.123
ThreadID: 8
ConnectionID: 42
PacketType: 0x03
ByteCount: 11
```

### Packet Type Reference

| Type | Hex | Description |
|------|-----|-------------|
| SendBodyChunk | 0x03 | Response body data |
| SendHeaders | 0x04 | Response headers + status |
| EndResponse | 0x05 | End of response |
| GetBodyChunk | 0x06 | Request more POST body data |
| CPongReply | 0x09 | Reply to CPing |
| ForwardRequest | 0x02 | Outgoing request to Tomcat |
